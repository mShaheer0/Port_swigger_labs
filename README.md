## 📝 Writeup Format

Each writeup documents the following structure:

1. **Vulnerability** — Description of the security flaw.
2. **Steps to Reproduce** — Clear, exact steps taken to exploit the vulnerability.
3. **Key Insight** — The “aha!” moment or main takeaway.
4. **Impact** — Real-world risk or consequence if exploited.
5. **Remediation** — Guidance for developers to fix or mitigate the issue.
6. **Tools Used** — Tools and resources leveraged during exploitation.
